# 小红书搜索 🔍

浏览器自动化搜索小红书笔记，支持关键词搜索、结果提取、筛选排序。

## 前置条件

- 浏览器已启动并运行
- 用户已登录小红书（Cookie 有效）
- 网络连接正常

---

## 功能：搜索笔记

### 操作步骤

1. **导航到小红书首页**
   ```
   navigate: https://www.xiaohongshu.com
   ```

2. **等待页面加载**
   ```
   wait_for: 2-3 秒
   ```

3. **定位搜索框**
   - 特征：`input[placeholder="搜索小红书"]`
   - 位置：页面顶部导航栏中央

4. **输入搜索关键词**
   ```
   type: 搜索框输入关键词
   ```
   - 支持任意主题：展览、美食、旅行、穿搭等

5. **提交搜索**
   ```
   press_key: Enter
   ```
   或点击搜索按钮（放大镜图标）

6. **等待结果加载**
   ```
   wait_for: 2-3 秒
   ```

7. **获取搜索结果**
   ```
   snapshot: 获取页面内容
   ```

### 结果提取

从搜索结果中提取以下信息：

| 字段 | 说明 | 示例 |
|------|------|------|
| 标题 | 笔记主标题（≤20字） | "深圳摄影展攻略" |
| 作者 | 发布者昵称 | "摄影达人小王" |
| 点赞数 | 互动数据 | "1.2万" |
| 封面图 | 首张图片 URL | 图片链接 |
| 笔记链接 | 详情页 URL | /explore/xxx |

### 排序方式

搜索结果支持切换排序：

- **综合**（默认）：综合相关度和热度
- **最新**：按发布时间倒序
- **最热**：按互动数据排序

点击对应标签切换排序方式。

## 使用示例

### 示例 1：基础搜索

用户说："搜索小红书上的摄影展内容"

```
1. navigate: https://www.xiaohongshu.com
2. wait_for: 3
3. type: 搜索框 "摄影展"
4. press_key: Enter
5. wait_for: 3
6. snapshot: 获取结果
7. 提取前 5-10 条笔记信息
```

### 示例 2：筛选最新内容

用户说："搜索最新的深圳美食推荐"

```
1. navigate: https://www.xiaohongshu.com
2. type: 搜索框 "深圳美食推荐"
3. press_key: Enter
4. wait_for: 3
5. click: "最新" 标签
6. wait_for: 2
7. snapshot: 获取结果
```

---

## 注意事项

- **频率控制**：每次搜索间隔 5 秒以上
- **结果数量**：建议每次获取 5-10 条结果
- **反爬机制**：避免短时间内频繁搜索
- **关键词**：支持中文、英文、混合搜索

## 故障排查

| 问题 | 原因 | 解决 |
|------|------|------|
| 搜索无结果 | 关键词太冷门 | 尝试更通用的关键词 |
| 页面加载慢 | 网络问题 | 增加 wait_for 时间 |
| 搜索结果为空 | 未登录 | 检查登录状态 |

# 小红书用户主页 👤

浏览器自动化查看用户主页，获取用户信息、笔记列表、统计数据。

## 前置条件

- 浏览器已启动并运行
- 用户已登录小红书
- 知道目标用户的 ID 或笔记中有该用户
- 网络连接正常

---

## 功能：查看用户主页

### 操作步骤

1. **访问用户主页**
   ```
   navigate: https://www.xiaohongshu.com/user/profile/<user_id>
   ```
   或从笔记中点击用户头像/昵称

2. **等待页面加载**
   ```
   wait_for: 2-3 秒
   ```

3. **获取用户信息**
   ```
   snapshot: 获取页面内容
   ```

### 信息提取

从用户主页提取以下信息：

| 字段 | 说明 | 示例 |
|------|------|------|
| 昵称 | 用户显示名称 | "摄影达人小王" |
| 小红书号 | 唯一标识 | @xiaohongshu_id |
| 简介 | 个人介绍 | "爱摄影爱生活" |
| 关注数 | 关注的人数 | "128" |
| 粉丝数 | 粉丝数量 | "1.2万" |
| 获赞数 | 总获赞数 | "5.6万" |
| 笔记列表 | 发布的笔记 | 笔记卡片列表 |

---

## 浏览用户笔记

### 操作步骤

1. **滚动浏览笔记列表**
   ```
   scroll: 向下滚动
   ```

2. **点击笔记查看详情**
   ```
   click: 笔记卡片
   ```

3. **返回用户主页**
   ```
   navigate_back
   ```

---

## 使用示例

### 示例 1：查看指定用户

用户说："查看用户 xxx 的主页"

```
1. navigate: https://www.xiaohongshu.com/user/profile/<user_id>
2. wait_for: 3
3. snapshot: 获取页面内容
4. 提取用户信息：昵称、简介、统计数据
5. 提取笔记列表（前 5-10 条）
```

### 示例 2：从笔记进入用户主页

用户说："看看这篇笔记的作者"

```
1. navigate: 笔记详情页
2. click: 作者头像或昵称
3. wait_for: 3
4. snapshot: 获取用户主页信息
```

---

## 注意事项

- **隐私设置**：部分用户可能设置了隐私权限
- **笔记数量**：新用户可能笔记较少
- **加载方式**：笔记列表通常是懒加载，需要滚动加载更多

---

## 故障排查

| 问题 | 原因 | 解决 |
|------|------|------|
| 页面不存在 | 用户 ID 错误 | 检查用户 ID |
| 无法查看 | 隐私设置 | 该用户设置了隐私 |
| 笔记加载慢 | 网络问题 | 增加等待时间 |

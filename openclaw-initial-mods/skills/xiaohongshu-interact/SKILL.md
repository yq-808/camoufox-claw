# 小红书互动 💬❤️⭐

浏览器自动化进行点赞、收藏、评论、回复等互动操作。

## 前置条件

- 浏览器已启动并运行
- 用户已登录小红书
- 知道目标笔记的 URL 或已找到笔记
- 网络连接正常

---

## 功能 1：点赞笔记 ❤️

### 操作步骤

1. **访问目标笔记**
   ```
   navigate: https://www.xiaohongshu.com/explore/<note_id>
   ```

2. **定位点赞按钮**
   - 特征：心形图标 ❤️
   - 位置：笔记底部或右侧
   - 未点赞：空心图标
   - 已点赞：实心/红色图标

3. **执行点赞**
   ```
   click: 心形图标
   ```

4. **确认成功**
   - 图标变为实心/红色
   - 点赞数 +1

### 取消点赞

再次点击已点赞的心形图标，图标恢复空心。

---

## 功能 2：收藏笔记 ⭐

### 操作步骤

1. **访问目标笔记**
   ```
   navigate: 笔记详情页
   ```

2. **定位收藏按钮**
   - 特征：星形图标 ⭐
   - 位置：点赞按钮旁边
   - 未收藏：空心图标
   - 已收藏：实心/黄色图标

3. **执行收藏**
   ```
   click: 星形图标
   ```

4. **确认成功**
   - 图标变为实心/黄色
   - 可选择收藏夹（如有多个）

### 取消收藏

再次点击已收藏的星形图标。

---

## 功能 3：发表评论 💬

### 操作步骤

1. **访问目标笔记**
   ```
   navigate: 笔记详情页
   ```

2. **滚动到评论区**
   ```
   scroll: 页面底部
   ```

3. **定位评论输入框**
   - 特征：`textarea[placeholder*="说点"]`
   - placeholder 可能为："说点什么..."、"发表评论"

4. **输入评论内容**
   ```
   type: 评论框 "你的评论内容"
   ```
   - 支持 emoji
   - 可以 @ 用户

5. **发送评论**
   ```
   click: "发送" 按钮
   ```

### 评论技巧

- 真诚分享经验或感受
- 可以提问引发互动
- 避免广告和引流内容
- 适当使用 emoji 增加亲和力

---

## 功能 4：回复评论 ↩️

### 操作步骤

1. **访问笔记详情页**
   ```
   navigate: 笔记详情页
   ```

2. **找到目标评论**
   ```
   scroll: 在评论区查找
   ```

3. **点击回复**
   ```
   click: "回复" 按钮
   ```
   或点击评论内容本身

4. **输入回复内容**
   ```
   type: 回复框 "你的回复"
   ```

5. **发送回复**
   ```
   click: "发送" 按钮
   ```

## 使用示例

### 示例 1：点赞笔记

用户说："给这篇笔记点赞"

```
1. navigate: 笔记详情页 URL
2. click: 心形图标（点赞按钮）
3. 确认图标变为实心
```

### 示例 2：评论笔记

用户说："评论'拍得很好'"

```
1. navigate: 笔记详情页
2. scroll: 到评论区
3. click: 评论输入框
4. type: "拍得很好"
5. click: "发送" 按钮
```

### 示例 3：点赞并收藏

用户说："点赞并收藏这篇笔记"

```
1. navigate: 笔记详情页
2. click: 心形图标（点赞）
3. click: 星形图标（收藏）
4. 确认两个图标都变为实心
```

---

## 注意事项

- **频率控制**：评论间隔 30 秒以上
- **内容合规**：避免敏感词和广告
- **自然频率**：点赞收藏不要过于频繁
- **互动质量**：真诚评论更容易获得回复

---

## 故障排查

| 问题 | 原因 | 解决 |
|------|------|------|
| 点赞失败 | 已点赞 | 检查图标状态 |
| 评论发送失败 | 内容为空 | 检查输入内容 |
| 找不到评论框 | 页面未加载 | 等待页面加载完成 |
| 操作被拒绝 | 过于频繁 | 等待后再试 |
